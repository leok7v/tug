Docker's a container ship. tug's a tugboat.

Containers need a system-wide daemon, namespaces, cgroups, storage layers and root. tug is a single unprivileged file: a tiny kernel, toybox, and your payload — all in one.

Small room, no elephants.

Everything a developer agent needs, nothing it doesn't.

📦

One binary

BIOS, a Linux 6.1 kernel and an interactive rootfs are .incbin'd into one self-contained executable. Run it from anywhere, no install.

🔓

No daemons, no root

Pure user-mode emulation on Fabrice Bellard's TinyEMU. Nothing system-wide, nothing privileged — ideal where runtime code-gen is forbidden (iOS / Android).

🧰

Real toolchains, on demand

Alpine apk on a persistent disk: essentials installs build-base, clang, node + npm, python + pip, rust. Downloaded when you want them.

🌐

Networking & HTTPS

slirp user-mode NAT — the guest reaches the Internet with no host privileges. curl + mbedTLS for TLS, a CA bundle baked in.

🔗

Share files & SSH in

-S dir mounts a host folder over 9p at /mnt/share; -L 2222:22 + tug-sshd gives you a real SSH login.

⚡

Tiny & fast

~6.5 MiB fixed emulator overhead, boots in as little as 32 MiB, sub-0.2 s to userspace. A full modern Linux sandbox in ~38 MiB.

Up and running in seconds.

Build the self-contained binary, boot it, and you're at a shell. Add a persistent disk and apk brings in real toolchains.

# build a single self-contained binary make embed-apk # create a sparse 32G persistent disk (apk userland) make disk # boot it — no args, no daemons ./tug-embedded-apk tug:~# essentials # install clang / node / python ... tug:~# apk add neovim ripgrep tug:~# clang hello.c -o hello && ./hello hello, world
